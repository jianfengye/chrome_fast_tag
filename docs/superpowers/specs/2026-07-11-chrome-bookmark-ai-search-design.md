# Chrome 智能书签定位 — 设计规格

日期：2026-07-11  
状态：已定稿（待实现计划）

## 1. 问题与目标

浏览器书签过多时，用户往往只记得「内容语义」（例如「讲 React 性能的那篇」），而书签标题/URL 用词对不上，Chrome 自带书签管理器无法按语义找到。

本产品做一个 Chrome 扩展：用快捷键唤起搜索框，先对全部书签做本地模糊匹配秒出结果；若用户配置了 DeepSeek API Key，再异步用大模型做语义联想，把相关书签插到列表前部。选中后：若已有同 URL 标签页则激活，否则新开。

## 2. 用户与成功标准

- **用户**：个人重度书签用户（自用优先）
- **成功标准**：
  - 输入后本地结果在约 100ms 内出现
  - 配置 Key 后，标题对不上记忆的书签能被 AI 捞到并靠前展示
  - 一次回车到达正确页面（激活已有标签或新开）

## 3. 方案选择

采用 **扩展内「本地模糊 + DeepSeek 对话式匹配」**：

| 方案 | 结论 |
|------|------|
| A. 本地 Fuse + DeepSeek 返回 id 列表 | **采用（MVP）** |
| B. 本地向量 / Embedding 索引 | 书签量大时的 v2 |
| C. 仅对本地 Top N 做 AI 重排 | 召回不足，不满足核心痛点 |

## 4. 架构

Chrome Extension Manifest V3，四个模块：

1. **Search Overlay** — 快捷键唤起的居中搜索 UI（类 Spotlight）
2. **Service Worker** — 书签缓存、本地检索、调用 DeepSeek、打开/激活标签
3. **Settings（Popup/Options）** — 配置 API Key、模型、快捷键说明
4. **本地存储** — `chrome.storage.local` 存 Token 与偏好；书签运行时从 `chrome.bookmarks` 读取并短缓存

### 主数据流

1. 快捷键 / 扩展图标 → 打开 Overlay  
2. 读取书签树 → 本地模糊匹配 → **立即展示**  
3. 若有 Token → 防抖后调 DeepSeek → 按返回 id **混排插入**并标「AI」  
4. 选中 → 查 `chrome.tabs` 是否已有同 URL → 有则 activate，无则 create  
5. 设置页保存 DeepSeek API Key  

### 权限

`bookmarks` · `tabs` · `storage` · `commands` · DeepSeek API 对应 host 权限

## 5. 搜索体验

- 唤起：全局快捷键为主；扩展图标兼做入口与设置
- 覆盖范围：全部书签（含各文件夹）；结果展示标题、文件夹路径、URL
- 结果呈现：**混排** — 本地结果先出；AI 返回后将语义相关项插入列表前部并标「AI」，已出现的不重复
- 键盘：`↑↓` 选择，`↵` 智能打开，`Esc` 关闭；可选 `⌘/Ctrl+↵` 强制新开
- 无 Token：仅本地搜索，UI 可轻提示「配置 Key 启用 AI」

## 6. DeepSeek 匹配策略

1. 输入防抖约 **300ms** 后，若已配置 Token，发起 AI 请求  
2. 载荷：用户查询 + 书签条目（`id`、标题、URL、文件夹路径）  
3. 书签很多时的裁剪：
   - 优先送本地模糊 Top N（如 80）
   - 本地几乎无命中时，再送「标题 + 文件夹」的轻量全库摘要（截断到 token 预算内）
4. 模型只返回相关书签 **id 的有序 JSON 列表**，不写长解释  
5. 收到后混排插入；用户再次改输入则取消未完成请求并重跑  
6. 失败（超时 / 401 / 网络）：静默保留本地结果，轻提示「AI 暂不可用」  
7. 默认模型：`deepseek-chat`（设置可改）

隐私约定：调用时上传标题 + 完整 URL + 文件夹路径（用户已接受）。

## 7. 打开行为

- `↵`：去掉 URL hash 后比较；若已有同 URL 标签 → 激活该标签并聚焦窗口；否则新标签打开
- 查询参数参与比较（避免不同页面被误合并）
- `⌘/Ctrl+↵`：强制新开

## 8. 设置

- DeepSeek API Key → `chrome.storage.local`（不明文写入代码）
- 可选模型名（默认 `deepseek-chat`）
- 快捷键说明 + 链到 Chrome「扩展程序快捷键」
- 「测试连接」按钮
- Key 可选：无 Key 仍可用本地搜索

## 9. MVP 范围

### 包含

- Chrome MV3 扩展
- 快捷键 + 图标唤起 Overlay
- 全部书签本地模糊（标题 / URL / 文件夹）
- 可选 DeepSeek 异步混排联想
- 智能打开（同 URL 激活 / 否则新开）
- 设置页配置 API Key

### 不做（后续）

- 页面正文 / 摘要抓取
- 本地向量索引 / Embedding
- 最近关闭标签、历史记录搜索
- 书签增删改与文件夹管理
- 多模型厂商、账号体系、云同步
- Firefox / Safari

## 10. 错误处理（摘要）

| 场景 | 行为 |
|------|------|
| 无书签 / 空查询 | 空态提示 |
| 本地无命中且无 AI | 提示无结果；有 AI 则等待联想 |
| AI 失败 | 保留本地结果 + 轻提示 |
| Key 无效 | 测试连接失败提示；搜索时降级本地 |
| DeepSeek 限流 | 提示稍后重试，不阻塞本地 |

## 11. 测试要点

- 本地模糊：中英文、文件夹路径、部分 URL
- 混排：AI id 去重、插入顺序、加载态
- 打开：已开同 URL（含 hash 差异）、未开、强制新开
- 无 Key / 错误 Key / 断网降级
- 书签变更后缓存刷新

## 12. 已确认决策摘要

- 对象是**书签**，不是标签页  
- 痛点：标题对不上记忆 → 语义联想  
- MVP 只用标题 + URL（+ 文件夹）；摘要后续  
- 本地先出，AI 异步补  
- 搜全部书签  
- Token 可选  
- 上传标题 + URL + 文件夹  
- 结果混排并标「AI」  
- 已开则激活，否则新开
